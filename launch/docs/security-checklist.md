# Security Setup — The Pack Guys

> Threats to a new e-commerce business: domain hijacking, email takeover (→ password resets everywhere), supplier-side phishing (fake invoices from your "China friend"), Shopify admin compromise, payment-processor fraud, customer PII breach. Below is the practical hardening sequence — ordered by impact.

---

## Tier 1 — Do TODAY (15 min, $0)

These are the highest-leverage security wins. Skip nothing.

### 1. Enable 2FA on Gmail (lamberthahm@gmail.com)
- Gmail is the master key — password resets for everything flow through it
- **Settings → Security → 2-Step Verification → ON**
- Method preference: **Passkey** (Touch ID on your Mac, biometric on iPhone). Far better than SMS.
- Backup method: Authenticator app (1Password or Google Authenticator), NOT SMS — SMS is vulnerable to SIM swap
- Generate + save **10 backup codes** in 1Password vault

### 2. Enable 2FA on Porkbun
- **Account → Security → Two-Factor Authentication → ON**
- TOTP (authenticator app) — store in 1Password
- Porkbun controls your domain — losing this = losing the brand. **Critical.**

### 3. Enable 2FA on GitHub (Meatballmonster)
- **Settings → Password and authentication → Two-factor authentication → Enable**
- Method: TOTP via 1Password
- GitHub holds your deployed site source — compromise = site defacement / malicious code injection

### 4. Review Gmail's "Apps with access" list
- **myaccount.google.com → Security → Third-party apps with account access**
- Remove anything you don't recognize
- These are old OAuth grants that may still have access — silent risk

### 5. Verify Porkbun domain lock is ON
- Already verified earlier in setup — but confirm: **Domain Management → thepackguys.com → Details → "Domain Lock: ON"**
- Prevents transfer to another registrar without your manual unlock

## Tier 2 — This Week (30 min total, $0)

### 6. Add email forwarding aliases instead of giving out your real Gmail
- Set up in Porkbun: `sales@`, `help@`, `hi@`, `admin@`, `billing@`, `legal@` all forwarding to `lamberthahm@gmail.com`
- **Hand out role-based aliases publicly, never your real Gmail**
- If `sales@thepackguys.com` gets spammed/compromised in a customer's CRM, you can kill the forward without losing your real email

### 7. Add SPF / DKIM / DMARC records for the domain
- These prevent spoofing of `@thepackguys.com` email (someone faking emails AS your brand)
- Porkbun already added a basic SPF record (`v=spf1 include:_spf.porkbun.com ~all`) — visible in your DNS already
- DKIM is added automatically by Porkbun email forwarding
- **Add DMARC manually:** `_dmarc.thepackguys.com TXT v=DMARC1; p=quarantine; rua=mailto:admin@thepackguys.com`
- I can add this via Porkbun API in one command

### 8. Use 1Password for every credential going forward
- Already in your app stack — start a "The Pack Guys" vault
- Every password generated by 1Password (long, random, unique)
- Never reuse passwords across services
- Store: Porkbun, GitHub, Mercury (when opened), Stripe, Shopify, China supplier portal, Amazon Seller Central, USPTO account, CDTFA login

### 9. Set up Gmail filter for security alerts
- Create a label `security-alerts`
- Filter rule: from `noreply@github.com`, `support@porkbun.com`, `account-security@*` → label + star + never spam
- Catch breach notifications, login alerts, password reset attempts immediately

## Tier 3 — Before Real Revenue (1 hour, $50–150)

### 10. Hardware security key (YubiKey 5 NFC — ~$50)
- Phishing-resistant 2FA for the highest-value accounts
- Required on: Gmail (master key), Porkbun, GitHub, Mercury, Stripe
- Order on Amazon — ships same day. **The single best security purchase you can make.**

### 11. Business email forwarding hygiene
- Once Shopify is live, set `Shopify admin notifications → admin@thepackguys.com` (not your personal Gmail)
- Same for Stripe alerts, Mercury alerts
- Compartmentalizes: if `admin@` is compromised, your personal Gmail still secure

### 12. China supplier — anti-phishing protocol
- Real risk: someone impersonates your supplier with a fake invoice asking for wire to a new bank
- **Hard rule: any change to wire instructions requires a video call with him to confirm.** Always. No exceptions.
- Save his known bank details + SWIFT code in 1Password
- Never wire to "updated" bank info without that video call

### 13. Customer payment security
- **Never enter credit cards or bank details into customer-facing forms** — Stripe handles all PCI compliance, you should never touch raw card data
- If a customer emails you their card "to make it easier," reply asking them to use the secure Stripe checkout link
- Your hands-off-card-data posture is the legal shield against PCI liability

## Tier 4 — Ongoing Discipline (10 min/month)

### 14. Monthly security review (set a calendar reminder)
- Check all "Apps with access" lists (Gmail, GitHub, Porkbun, Shopify) — remove anything unused
- Review active sessions on each (kill any you don't recognize)
- Rotate API keys quarterly (Porkbun, Stripe test keys, etc.)
- Review who has admin access to your accounts

### 15. Backup the GitHub repo locally
- Run weekly: `cd ~/business/preroll-tubes && git clone https://github.com/Meatballmonster/thepackguys backup-$(date +%Y%m%d)`
- If GitHub goes down or repo gets compromised, you have a local copy

### 16. Watch out for these specific scams targeting new e-commerce businesses
- **Fake "trademark filing" emails** claiming someone is trademarking your name in another country — extortion scam, ignore
- **Fake Google verification calls** asking to "verify your business" — Google doesn't call, ignore
- **Fake Shopify suspension notices** asking you to log in via a link — phishing, always go to shopify.com directly
- **Fake supplier "second invoice"** for a payment already made — confirm via known channel
